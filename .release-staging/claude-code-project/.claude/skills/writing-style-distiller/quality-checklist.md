# 质检清单 (Quality Checklist — v2)

在产出最终 `<writing_style>` 文件之前，必须完成以下 5 维质检。

## A. 规则抽象度（反 AI 场景列举偏置）

这是最关键的检查维度。

- [ ] 所有规则是否是以动词开头的、对「一类写作场景手法」的抽象归类？
- [ ] 绝无具体场景枚举词（如：写砧板切菜声、写电热壶水汽糊住眼睛）——具体场景只允许出现在 `>` 示例中
- [ ] 无解释性短语（如："因为读者需要感受到..."、"这是为了保证..."）——删除解释，保留指令
- [ ] 一句话概括测试通过：每条规则的展开/例证是否能被规则本身概括？如果能，删除展开
- [ ] 无不必要的概念定义：模型已知的类型名（如"温情生活流""意识流"）不再解释其含义
- [ ] 无重复目标声明——同一个目标只在一个位置出现
- [ ] 每条规则不超过两句，最好一句动词开头

## B. 格式规范度

- [ ] 最外层被完整的 `<writing_style>` 标签包裹，属性完整（author, source, mode, family）
- [ ] 一级子标签仅含五大必选标签 + 可选 `<dynamics>`：`<control_scope>`、`<narrative>`、`<expression>`、`<aesthetics>`、`<prompt_discipline>`、`<dynamics>`（仅场景响应类/对话驱动类）
- [ ] `<dynamics>` 仅在场景响应类或对话驱动类文风中出现，其他家族不应包含此标签
- [ ] 内部没有任何 XML 子标签嵌套
- [ ] 原文示例皆被 `>` 引用标记包裹，保持原语种
- [ ] 无遗漏的分区标签

## C. 文风与剧情边界

- [ ] 无任何条款控制未来剧情走向、角色结局或关系发展
- [ ] progression 只描述单次回复内部的情感曲线或视角变化
- [ ] ending 只描述单次回复的语态收口，无情节结局暗示
- [ ] 如有剧情基调，确认隔离在独立 `<tone>` 块中

## D. AIRP 鲁棒性

- [ ] 所有负向约束后以 `→` 附带正向替代动作
- [ ] 极端词（必须、绝对、一定、强制）仅用于硬格式规则
- [ ] 已绑定目标模型特性适配规则（Claude 抑文青病 / Gemini 抑连锁反应）
- [ ] 重置规则覆盖 NSFW/特殊场景后的风格漂移恢复
- [ ] 示例不借用剧情、人物、场景序列

## E. 语言干净度

- [ ] 无 AI 无用赞美词：独特的、善于、富有、常用、巧妙地
- [ ] 无 AI 模式化修辞：让文字呼吸、编织情感、画下色彩、赋予温度
- [ ] 无装饰性副词和无功能性修饰

## F. 规则间动态一致性

- [ ] 多条规则叠加时是否存在模型无法解决的矛盾？（如同时要求"极致口语化"和"文言为主"）
- [ ] 刻意设计的张力是否有明确的优先级或共存声明？（如"身体适应"与"反抗意志"可以并存、"内心丰富"与"外在简洁"是刻意反差）
- [ ] 状态切换规则之间是否有清晰的互斥/并存/优先级声明？
- [ ] 同一概念是否在不同分区中被重复定义且定义不一致？

## G. 铁律有效性

- [ ] 铁律级别的禁令是否与一般性偏好做了明确区分？（如用 `【禁止】` `【铁律】` 等标记与普通规则区分）
- [ ] 负向约束是否足够具体，不至于模型理解为"减少使用"而非"完全禁止"？
- [ ] 铁律是否独立成段或有醒目标记，而非淹没在大段规则中？
- [ ] 每条铁律是否提供了正向替代方案（禁止X → 改用Y），避免模型只知道不做什么却不知道做什么？

## H. 语域统一性（语言特征类文风必检）

- [ ] 虚词、实词、句法骨架是否属于同一语域层级？
- [ ] 是否存在现代白话句法结构混入古风/文言规则的情况？
- [ ] 是否存在翻译腔句骨（英文句法结构的中文表皮）？
- [ ] 正反例对中的"正确写法"是否真正达到了目标语域？
