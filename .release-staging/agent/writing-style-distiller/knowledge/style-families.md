# 风格家族与密度决策

本文件指导在 Phase 2 (Diagnosis) 中如何诊断文风类别，并进行密度分配。

## 1. 风格家族分类系统

### 1.1 情感表达类
- 主控轴：情感、读者体验
- 特点：加上类型标签（如"二次元"）AI 就能理解基础风格，应把 token 花在情绪处理、动作细节和段落节奏上。
- 适用场景：欢快有爱、温馨日常、伤感命运、治愈心灵。
- 典型代表：二次元文风, 治愈文风, 温情生活流。

### 1.2 语言特征类
- 主控轴：语言、词汇、语法
- 特点：AI 训练数据中的特殊古雅风格语料（如红楼、文言、民国白话）极度不稳定，容易穿帮。必须给极其详尽的词法句法规则，并辅以高频排除词。
- 适用场景：古风武侠、仙侠古典、明清通俗白话、特定方言/外语模仿。
- 典型代表：古风/文言文风, 红楼梦风, 方言文风。

### 1.3 叙事结构类
- 主控轴：视角、独白、结构
- 特点：核心考量为「人称与聚焦」和「意识流动节奏」。段落控制和叙述距离是核心手段。
- 适用场景：自传体、意识流文学、轻小说段落、群像转场。
- 典型代表：意识流, 散文小说化叙事, 自传体, 轻小说。

### 1.4 示例驱动类
- 主控轴：示例演示
- 特点：使用轻量通用指令框架，配合 1-2 段高质量整段示例来教节奏。更换示例即更换风格。整个 profile 依赖 examples，其他字段保持高度精简。
- 典型代表：通用散文文风 + 换示例切换风味。

#### 示例选择标准

示例是示例驱动类文风的核心资产，选择不当会污染整个文风。

**必须是完整场景**：
- 正确：一段包含对话、动作、叙述的完整段落（如《受戒》开篇、《我与地坛》第一段）
- 错误：几句零散的描写句，没有上下文

**token 参考量**：单段示例约 400 token，整个文风（框架+示例）不超过 1000 token 为佳。

**示例对基调有决定性影响**：示例的内容主题会强烈影响 AI 的输出基调。选择温暖日常的示例，AI 会倾向温暖日常的写法；选择阴暗压抑的示例，AI 会倾向阴暗压抑。必须意识到这一点并主动选择。

**人称与示例的匹配策略**：
- 第一人称：示例应体现叙述者的主观感受和个人特质，因为第一人称更需要体现角色/user 本身
- 第三人称：示例应体现讲述世界和故事的方式，因为第三人称的优势在于更自由地叙述

**框架应偏大众化**：
- 示例驱动类的规则框架不应过于专业或小众
- 可以使用中学语文的文本分析方法来构建：叙述视角、语言风格、表达方式、情感基调
- 核心是让框架能够适配不同示例，而不是锁死在某种特定风格上

#### 示例驱动类的通用框架参考

```
- 文体定义:
    - 「闲谈」不是做文章，呈现的是故事生活中的视觉语言
    - 「散文」不存在典型，结构松散，追求内在的意境
    - 「小说」是载体，产生具体的实感
- 文体特征:
    - 语言准确，畅通，平易，雅致
    - 关注生活的一处角落，一个片段
    - 不过分刻画人物，不描写/解读说话者的语气神态
```

注意：以上框架中的文体定义是针对"散文小说化叙事"这一特定风格的，此处保留是因为这类风格确实需要让模型理解其跨体裁特性。对于模型已知的常见类型名，不应重复定义。

### 1.5 场景响应类
- 主控轴：场景状态 → 行为响应
- 特点：核心不是静态的风格描述，而是一套「条件触发 → 写作行为切换」的动态规则系统。文风随场景状态（力度分级、情绪阶段、关系变化等）实时切换写法，本质是一个状态机。
- 适用场景：NSFW 场景写作指南、战斗系统描写规则、情绪阶段递进式文风、需要根据剧情状态切换笔法的复杂系统。
- 典型代表：NSFW 现实化指导、骚话文风（含温度层切换）、战斗描写分级系统。

#### 场景响应类的结构要素

1. **状态分级表**：明确定义各状态等级及其触发条件（如力度分级：较轻→正常→较大→破坏性）
2. **响应映射规则**：每个状态等级对应的具体写作行为（描写方式、用词范围、感官侧重）
3. **状态转换逻辑**：状态之间如何转换、是否可以并存、优先级如何
4. **机制间交互**：多个机制同时触发时的处理规则（如「身体适应机制」与「反抗机制」可以并存）

#### 与其他家族的关键区别

- 情感表达类的规则是**恒定的**（始终用某种方式写情感）
- 场景响应类的规则是**条件性的**（在 X 条件下用 A 方式写，在 Y 条件下用 B 方式写）
- 场景响应类的 `<prompt_discipline>` 密度通常很高，因为需要精确定义状态边界和转换条件

### 1.6 对话驱动类
- 主控轴：对话、声口、互动节奏
- 特点：对话是文本的骨架和主要推进力，叙述和描写围绕对话展开而非反过来。核心挑战在于让对话本身承载风格辨识度——通过声口、称呼系统、温度切换、对话与叙述的分离规则来实现。
- 适用场景：轻小说对话驱动、骚话文风、漫才式互动、角色扮演对话密集型文风。
- 典型代表：伏见司式青春喜剧、骚话文风（现代版/古风版）、男性视觉小说的对话互动部分。

#### 对话驱动类的核心维度

1. **声口系统**：角色说话的口吻、用词习惯、句式节奏——这是风格辨识度的核心载体
2. **称呼升级引擎**：称呼随情绪/关系升级而变化，是对话推进力的关键工具
3. **温度层切换**：同一场对话中在央求/放浪/调戏/假装抗拒等温度之间自由切换
4. **对话与叙述的硬分离**：对话独立成段、禁止对话标签、用动作承接对话等格式规则
5. **内外反差**（可选）：内心活动丰富 vs 外在表达简洁（如伏见司式的吐槽役结构）

### 1.7 人格化写法（可选路径）

人格化写法不是一个独立的风格家族，而是一种**可叠加到任何家族上的表述策略**。

- 核心思路：用作者/作品的口吻来撰写文风规则，将规则和语料合二为一
- 效果：对抑制八股有显著帮助，因为规则本身就带有作品的气质
- 难度：需要对作者风格有足够深入的理解，否则模仿不到位

#### 实施要点

1. **研究作者/作品的核心意象和表达方式**，用这些意象替代一般化的规则描述
2. **"作品的口吻"比"作者本人的话"更重要**——可以采纳作者原话，但不能生搬硬套，整体风格和逻辑要统一
3. **外国作品注意译者干扰**：翻译文本会混入译者风格，要分辨捕捉到的是译者还是作者。原语种关于作者的语料通常更丰富
4. **可能与预设的角色人格冲突**：文风规则的人格化口吻和角色的人格可能打架，需要补充隔离规则

#### 适用判断

- 当蒸馏对象的作品有极强的辨识度（如帕拉尼克、太宰治），人格化写法可以考虑
- 当用户对作者风格有深入研究并能提供充分素材时，可以建议此路径
- 不建议在用户对作者了解有限时使用——这种情况下规则化写法更可靠

## 2. 密度决策分配表

根据诊断出的风格家族，自适应调整 `<writing_style>` 最终产出里的字段密度：

| 风格家族 | <narrative> 密度 | <expression> 密度 | <aesthetics> 密度 | <prompt_discipline> 密度 | <dynamics> 密度 | 目标 Token 范围 |
|---|---|---|---|---|---|---|
| 情感表达类 | 中 | 高 (重人物心理与感官) | 高 (美学核心) | 中 | 无 | 800-1000 |
| 语言特征类 | 中 | 高 (重对白语法与描写) | 高 (重词汇句法细节) | 高 (标签转换边界) | 无 | 1000-1200 |
| 叙事结构类 | 高 (人称/段落/时序) | 高 (重独白与人物方法) | 低 | 中 | 无 | 800-1000 |
| 示例驱动类 | 低 | 中 (重示例引流) | 低 | 中 + 包含示例流模块 | 无 | 600-800 |
| 场景响应类 | 中 | 高 (重状态对应描写) | 中 | 高 (状态边界与转换) | 高 (状态机核心) | 1000-1400 |
| 对话驱动类 | 中 (重对话与叙述分离) | 高 (重声口/称呼/温度层) | 低 | 中 | 中 (称呼升级/温度切换) | 800-1100 |

## 3. 密度等级定义

- **高密度**：展开 4-6 个具体的指令项，每个子项附带 1 个 30-80 字符的例句引用。
- **中密度**：覆盖 2-3 个核心指令，仅在关键特征附带例句。
- **低密度**：合并为一个摘要句段，不带例句，不展开子项。
