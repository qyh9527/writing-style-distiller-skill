# Phase 2: 风格诊断与架构设计

本阶段根据 Phase 1 的研究笔记和用户反馈，确定风格家族、控制轴、密度分配和选段规划。

## 1. 风格家族精确核定

读取 `knowledge/style-families.md`，将本案例精确归类：
- 情感表达类
- 语言特征类
- 叙事结构类
- 示例驱动类

归类依据：主控轴是什么？用户最在意的表达维度在哪里？

## 2. 密度分配矩阵设计

根据确定的家族，从 `knowledge/style-families.md` 的密度决策表中取出对应行，明确写好 4 个分区标签的密度等级和子项规划。

## 3. 特殊机制规划

- **标签风险扫描**：分析研究笔记中有哪些高风险标签（如傲娇、破碎、温柔、压抑），参照 `knowledge/label-risk-table.md` 标记待转换项。
- **模型适配预判**：若用户指定了目标模型，参照 `knowledge/model-adaptation.md` 预规划 Phase 4 的适配策略。
- **示例流决策**：判断是否需要采用示例流策略（通常用于示例驱动类或 Gemini 目标模型）。

## 4. 选段规划

从研究笔记的摘录候选中，为每个需要 `>` 示例的字段选定最佳匹配：
- narrative 节奏/段落 → 选自 [来源]
- expression 描写/对话 → 选自 [来源]
- aesthetics 语言 → 选自 [来源]

## 6. 🚪 审查门

诊断书落盘后，展示密度分配方案和选段匹配，向用户提问：

> "风格诊断书已写入。家族定位为 [X]，密度配比为 [摘要]，已锁定 [N] 个原文匹配点。是否继续生成文风初稿？"

等待用户确认后才进入 Phase 3。
