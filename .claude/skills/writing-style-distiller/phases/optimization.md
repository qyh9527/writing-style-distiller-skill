# Phase 4: 提示词优化与最终质检

本阶段吸收用户的修改反馈，执行压缩优化和质检守卫，产出最终的 `<writing_style>` 配置。

## 1. 压缩优先级

按以下顺序裁剪初稿——先删低价值内容：

| 优先级 | 裁剪动作 |
|---|---|
| P1 | 删除解释性话段（"因为读者需要..."、"这是为了确保..."）。保留指令，删除理由。 |
| P2 | 合并重复目标声明。同一目标只在一处出现。 |
| P3 | 剔除无功能修饰词（非常、十分、truly、extremely、独特的、善于、富有）。 |
| P4 | 将多句规则转为单句动词开头指令。 |
| P5 | 每个字段保留最强的一个示例，删除较弱的重复示例。 |
| P6 | 保留具体标记词、段落规则、对话规则。这些优先于美学形容词。 |

Token 效率原则：一个精选示例的教学效果超过三句模糊描述。裁剪时优先保留示例，而非规则解释。

## 2. 标签与模型适配注入

- 读取 `knowledge/label-risk-table.md`，将初稿中残留的高风险标签替换为行为化规则。
- 读取 `knowledge/model-adaptation.md`，根据用户目标模型微调 `<prompt_discipline>`。
- 确保每个负向约束后有 `→` 正向替代。
- 移除不必要的极端词（必须、绝对、强制），仅保留硬格式规则中的极端词。

## 3. 质检守卫门

运行 `quality-checklist.md` 中的全部检查项：
- A. 规则抽象度
- B. 格式规范度
- C. 文风与剧情边界
- D. AIRP 鲁棒性
- E. 语言干净度

若检查到违规，立即修正后再输出。

## 4. 最终输出

按以下顺序在终端渲染（或保存为 `final-style.md`）：

1. **最终文风配置** — `<writing_style>` 在代码块内。
2. **独立 `<tone>` 块**（仅在用户请求时）。
3. **引用出处与选段理由**：
```text
【引用出处】
[1] 来源信息
[2] ...

【选段理由】
narrative.节奏：选自 [来源]，展现 [特征]。
expression.对话：选自 [来源]，展现 [特征]。
...
```
4. **质检记录**：
```text
【质检记录】
初稿长度估计：
问题（若有）：
修正操作：
终稿长度估计：
```

## 5. 证据诚实标签

在引用出处中，按需使用以下标签：
- `基于有限资料` — 可靠证据稀缺。
- `译文可得，原文未核验` — 有译文但原文未验证。
- `用户样本文本` — 证据来自用户提供的文本。
- `综合风格描述` — 配置文件是合成的，未绑定单一作者。
- `示例流演示` — 样本用于教学流程，不作为剧情证据。
- `模型适配规则` — 规则基于目标模型行为而非源文本证据。
