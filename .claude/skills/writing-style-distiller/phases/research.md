# Phase 1: 研究与证据收集

本阶段负责根据用户输入进行联网深度研究或源文档剖析，收集原文样本和文学分析证据。

## 0. 意图探针（路由完成后、正式研究前）

路由确定深度模式后，向用户提出 1-2 个轻量问题以锚定蒸馏方向，避免"不清楚自己要什么"导致的方向偏离。

探针问题池（按需选取，不必全问）：
- "你希望这个文风用在什么类型的创作上？"（确定应用场景：网文/轻小说/散文/AIRP等）
- "你脑海里有没有一段让你觉得'就是这种感觉'的文字？"（锚定审美基线）
- "你最想避免的文字效果是什么？"（反向锚定，排除不想要的方向）
- "你更看重哪个维度：文字的结构方式，还是文字带给读者的感受？"（定位文字结构 vs 文字感受的侧重）

规则：
- 若用户输入已经非常明确（如指定了作者+代表作+目标模型+应用场景），可以跳过探针直接进入研究。
- 探针回答与路由诊断一起展示，不单独设审查门。
- 探针结果记录在研究笔记中，作为后续诊断的参考。

## 1. 深度模式：联网搜索规则（作者名输入时）

### 1.1 权威渠道优先级

搜索时**必须**按以下渠道优先级顺序选取证据来源，优先使用高可信度渠道：

| 优先级 | 渠道类型 | 示例 | 可信度 |
|---|---|---|---|
| ★★★ | **学术数据库与论文** | 知网(CNKI)、万方、Google Scholar、JSTOR、豆瓣读书学术书评 | 最高 |
| ★★★ | **作者原作全文/选集** | 正规出版社电子版、维基文库、Project Gutenberg、青空文库 | 最高 |
| ★★☆ | **权威文学批评与评论** | 文学期刊评论、知名学者/批评家文章、大学文学课程资料 | 高 |
| ★★☆ | **高质量百科与文学辞典** | 维基百科(有引用的条目)、文学术语辞典、作家年谱 | 高 |
| ★☆☆ | **社区深度分析** | 豆瓣长评(高赞)、知乎专栏(署名作者)、文学论坛深度帖 | 中 |
| ☆☆☆ | **自媒体与短内容** | 微信公众号、短视频文案、无来源博客 | 低——仅作补充，不可作为唯一来源 |

**渠道选取规则**：
- 每次研究至少使用 **2 个 ★★★ 级渠道** 的证据。
- ★☆☆ 及以下渠道的内容，必须与更高级渠道交叉验证后方可采信。
- 若搜索结果只有低可信度来源，应在研究笔记中标注 `基于有限资料`，并建议用户提供原文。
- 对于**非中文作者**，优先使用该作者母语的学术资源与原文，其次使用英文学术资源，最后使用中文翻译/评论。

### 1.2 搜索策略

进行至少 3-5 轮搜索，方向包含：
1. `[作者/作品] 代表作 文风 语言特征 分析`
2. `[作者/作品] 原文 摘录 选段`（必须收集原语种摘录，严禁二传机翻）
3. `[作者/作品] 叙事学 文本批评 论文`（抓取批评学者总结的审美术语）
4. `[作者] 文化美学 传统术语`（如：物哀、陰翳、間等源语言概念）
5. 若作者使用非英语语言，用该语言重复关键搜索。

提取目标：
- 至少 6 段原语种摘录候选。
- 覆盖叙事、表达、美学三大系统的证据。
- 批评术语和文化美学概念。

## 2. 标准模式：文本分析规则（本地文件/摘录输入时）

以用户提供的文本为首要证据来源，不要用外部百科概念覆盖文本本身的表现。

提取指标：
- 对白排版：对白与旁白如何分离？对白是否单独成段？
- 句式特征：流水散句还是密集排比？平均句长倾向？
- 核心情绪与人称视角。
- 段落控制模式：段落长短、情绪转换时的段落变化。
- 感官层级：优先使用哪些感官？

### 2.1 语法层级分析（语言特征类文风必做）

对语言特征类文风，需在标准提取指标之上增加四层分析：

1. **语音层面**：节奏韵律、句式长短交错规律、是否有韵文穿插
2. **词汇层面**：虚词来源（上古/中古/近古/现代）、实词的双音节化程度、是否有特定时代的生活用词
3. **语法层面**：主语省略习惯、定语小句使用频率、流水句 vs 复合句的比例、指示词使用习惯（如"那+名词"是否可省略）
4. **语用层面**：人物对话是否贴合身份（上层典雅 vs 下层俚俗）、情绪激烈处是否反用简笔

### 2.2 反面模式提取

在分析文本正面特征的同时，必须主动识别并记录「该风格下 AI 最容易犯的错误」：

- **该风格的 AI 典型失败模式是什么？**（如古风容易文白混杂、轻小说容易丢失吐槽节奏、NSFW 容易情绪通胀）
- **该风格有哪些"绝对不能做"的事？**（从用户提供的文本或案例中提取铁律，如"严禁情绪通胀""禁止文学替代词"）
- **该风格最容易被哪种八股侵蚀？**（参照 `knowledge/anti-bagu.md`，预判需要重点防御的八股类型）

记录格式：
```
## 反面模式预警
- AI典型失败：[具体描述]
- 铁律候选：[从文本中提取的绝对禁令]
- 八股高危项：[需要重点防御的anti-bagu编号]
```
