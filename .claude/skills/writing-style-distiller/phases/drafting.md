# Phase 3: 文风初稿生成

本阶段是核心生成阶段。严格依照 `output-format.md` 的格式规范和诊断书的密度分配，生成完整的 `<writing_style>` 初稿。

## 1. 核心书写约束

### 1.1 抽象优先原则（Primary Principle）

**强文风 prompt 说「如何写一类场景」，不说「写哪个场景」。**

每条规则的检验流程：
1. 规则是否命名了具体对象/事件/场景？→ 移到 `>` 示例或删除
2. 规则是否解释自身重要性？→ 删除解释，保留指令
3. 规则能否用一句动词开头的句子表达？→ 改写
4. 规则覆盖一类场景还是只覆盖样本中出现的那个？→ 拓宽到类

### 1.2 格式执行

- 使用 `output-format.md` 定义的 XML 壳 + 纯文本格式。
- 短动词句段，对于并列属性使用 `属性=值 | 属性=值` 缩短长度。
- 原文示例用 `>` 标记，紧跟所属规则，30-80 字符，保持原语种。

### 1.3 密度执行

- 按诊断书的密度矩阵分配内容量。
- 高密度分区：展开 4-6 个子项，每项附带示例。
- 中密度分区：覆盖 2-3 个核心规则。
- 低密度分区：压缩为 1-2 行摘要。

### 1.4 示例的高洁性保护（Plot-Borrowing Guard）

每个 `>` 下面的原文仅用来教 AI 认识「语气和节奏」：
- 保留：句法、词汇、修辞手法、节奏、段落形状、感官顺序、对话模式、聚焦方式。
- 移除或软化：具名对象、剧情序列、场景前提、角色关系、事件结果、世界观设定。
- 若具体对象同时是反复出现的象征意象，标记为可选母题而非必须内容。

## 2. 初稿产出

将完整的 `<writing_style>` 草案写入 `docs/style-output/{timestamp}-{style-name}/style-draft.md`。

## 3. 🚪 审查门

初稿落盘后，展示完整草案并向用户提问：

> "文风初稿已写入 `docs/style-output/{timestamp}-{style-name}/style-draft.md`。完整草案如下：[初稿]。请审查并告诉我哪些地方需要调整——特别关注段落控制、句法倾向、对话处理和负向约束部分。"

等待用户确认或提出修改意见后才进入 Phase 4。
