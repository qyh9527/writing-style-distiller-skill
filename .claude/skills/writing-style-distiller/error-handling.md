# 异常处理与常见错误

## 异常处理

| 情况 | 处理 |
|---|---|
| 原文摘录不足 | 标注 `基于有限资料`，减少示例到 4 段，说明限制。 |
| 多时期风格 | 在诊断书中标注分期，禁止将冲突风格揉在同一规则。 |
| 搜索失败 | 请求用户提供文本样本或代表作。 |
| 仅有风格描述 | 生成合成配置，标注 `综合风格描述`。 |
| 用户请求 AIRP 实用提示词 | 保持 `<writing_style>` 格式，增加模型适配和重置规则。 |
| 用户请求示例教学 | 在 XML 内用短摘录，长示例置于 XML 外并标注版权和剧情借用限制。 |
| NSFW 场景后风格漂移 | 在 `prompt_discipline` 中添加 NSFW 后重置规则，使用清爽的目标文风日常描写和独立短句重置基线。 |
| 预设与文风规则冲突 | 在 `control_scope` 中明确文风只控制"写作细节"和"段落控制"，不控制剧情走向；建议用户检查预设是否有重复指导（如"去八股""活人感"增强等条目）。 |
| 人格化写法与预设人格冲突 | 补充隔离规则，声明文风人格化口吻不代替角色人格；文风中的人格化口吻仅影响叙事语气，角色行为与性格以角色卡为准。 |

## 常见错误

- 不要伪造原文摘录。
- 不要翻译需要保持原语种的示例。
- 不要跳过引用和选段理由。
- 不要输出纯文学评论而无可执行写作参数。
- 不要让文风字段指挥未来事件、关系发展或结局。
- 不要用作者名替代具体写作规则。
- 不要将示例中的场景变为可复用的剧情素材。
- 不要依赖高风险标签（羞耻、疯感、破碎）而不附加行为化替换。
- 不要在规则中枚举具体场景替代抽象类规则。
- 不要未经质检就声称通过验证。
- 不要在文风中使用"物理温度""物理距离"等滥用物理限定词 → 直接写"温度""距离"。
- 不要将精液描述为"滚烫""炙热" → 精液微凉于体温。
- 不要在 NSFW 场景中写"触碰即发情"或"同步高潮" → 遵循生理逻辑。
- 不要让机制分级术语（如"身体适应机制""主要敏感区"）出现在小说正文中。
