# 风格家族与密度决策

本文件指导在 Phase 2 (Diagnosis) 中如何诊断文风类别，并进行密度分配。

## 1. 风格家族分类系统

### 1.1 情感表达类
- 主控轴：情感、读者体验
- 特点：加上类型标签（如"二次元"）AI 就能理解基础风格，应把 token 花在情绪处理、动作细节和段落节奏上。
- 适用场景：欢快有爱、温馨日常、伤感命运、治愈心灵。
- 典型代表：二次元文风, 治愈文风, 温情生活流。

### 1.2 语言特征类
- 主控轴：语言、词汇、语法
- 特点：AI 训练数据中的特殊古雅风格语料（如红楼、文言、民国白话）极度不稳定，容易穿帮。必须给极其详尽的词法句法规则，并辅以高频排除词。
- 适用场景：古风武侠、仙侠古典、明清通俗白话、特定方言/外语模仿。
- 典型代表：古风/文言文风, 红楼梦风, 方言文风。

### 1.3 叙事结构类
- 主控轴：视角、独白、结构
- 特点：核心考量为「人称与聚焦」和「意识流动节奏」。段落控制和叙述距离是核心手段。
- 适用场景：自传体、意识流文学、轻小说段落、群像转场。
- 典型代表：意识流, 散文小说化叙事, 自传体, 轻小说。

### 1.4 示例驱动类
- 主控轴：示例演示
- 特点：使用轻量通用指令框架，配合 1-2 段高质量整段小剧场示例来教节奏。更换示例即更换风格。整个 profile 依赖 examples，其他字段保持高度精简。
- 典型代表：通用散文文风 + 换示例切换风味。

## 2. 密度决策分配表

根据诊断出的风格家族，自适应调整 `<writing_style>` 最终产出里的字段密度：

| 风格家族 | <narrative> 密度 | <expression> 密度 | <aesthetics> 密度 | <prompt_discipline> 密度 | 目标 Token 范围 |
|---|---|---|---|---|---|
| 情感表达类 | 中 | 高 (重人物心理与感官) | 高 (美学核心) | 中 | 800-1000 |
| 语言特征类 | 中 | 高 (重对白语法与描写) | 高 (重词汇句法细节) | 高 (标签转换边界) | 1000-1200 |
| 叙事结构类 | 高 (人称/段落/时序) | 高 (重独白与人物方法) | 低 | 中 | 800-1000 |
| 示例驱动类 | 低 | 中 (重示例引流) | 低 | 中 + 包含示例流模块 | 600-800 |

## 3. 密度等级定义

- **高密度**：展开 4-6 个具体的指令项，每个子项附带 1 个 30-80 字符的例句引用。
- **中密度**：覆盖 2-3 个核心指令，仅在关键特征附带例句。
- **低密度**：合并为一个摘要句段，不带例句，不展开子项。
