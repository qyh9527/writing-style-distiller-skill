# 目标创作模型适配规则

本文件用于在 Phase 4 (Optimization) 中，根据用户的目标创作模型对文风提示词进行最终适配微调。

## 1. 通用规则

- 作者名是证据来源，不是最终指令。禁止使用"参考X"作为规则。
- 用具体特征替换作者引用：句长、转折词、聚焦方式、对话排版、感官层级。
- 示例不能驱动剧情内容。示例只教表达方式。

## 2. Claude 适配

### 2.1 Claude 的写作倾向与失败特征
- **同质精品化**：容易陷入"精致、舒缓、哀而不伤、优雅"的文青病 sameness。
- **过度阐释**：在旁白中解读每一个神态背后的宿命、情感纠葛、深层含意，造成正文虚浮。

### 2.2 针对性约束注入
- **具体声音约束**：强制注入类型特征（如轻小说的市井拌嘴、搞笑的日常吐槽）。添加"禁止主观抒情与作者介入评述，纯粹写动作与物理状态"。
- **句法反精致**：加重口语流水句比例，减少长对偶名词短语。在 `<expression>` 中增加"像提白菜价格一样平常地叙述大悲大喜"。
- **反过度解读**：呈现事实和姿态，不要解释每个情感含义。
- **XML 标签结构化**：Claude 4.x 系列对 XML 标签是最佳理解方式——`<writing_style>` 的分区标签格式本身就是 Claude 最佳实践。
- **禁止攻击性语言**：规则中不要使用 "CRITICAL!"、"YOU MUST"、全大写强调——对 Claude 来说这些反而导致更差结果。用平实的指令语气即可。

### 2.3 舒适区突破策略

Claude 的文青倾向（精致、舒缓、哀而不伤、优雅）已刻入底层权重，在舒适区内效果好，但舒适区外极难纠正——给语料示例、提关键词、亲自指导都无法让 Claude 写出轻松幽默搞笑的风格。

- **识别舒适区边界**：Claude 擅长散文、文学感悟、忧伤叙事、严肃文学。挣扎于：轻松搞笑、粗犷直白、市井泼辣、网文爽文。
- **非舒适区文风需多层约束叠加**：仅靠一两条规则无法把 Claude 拽出舒适区。需要在 `<expression>`、`<aesthetics>`、`<prompt_discipline>` 中同时施压。
- **示例密度提升**：对非舒适区风格，`>` 示例数量应增加到每区 2-3 个，用示例量压制 Claude 的默认回弹。
- **冷幽默/讽刺**需要特殊标注：Claude 即使写冷笑话也是文青式冷笑话。需要在规则中明确："幽默来自日常荒诞和人物反差，不来自文字的精巧。"

## 3. Gemini 适配

### 3.1 Gemini 的写作倾向与失败特征
- **极度极端化/连锁反应**：踩中一个标签容易一路疯滑。一写"害羞"直接变成瘫软崩溃。
- **过度重复提示词词汇**：容易在正文里复写提示词中提过的名词或短语。

### 3.2 针对性约束注入
- **行为化翻译**：彻底剥除情绪性标签。`<prompt_discipline>` 中必须是纯物理、纯语法规则。极少的性格描述，极大密度放在段落长短交割等排版约束上。
- **严格正向替代与边界隔离**：每个负向禁令后以 `→` 必须搭配正向替代动作。示例展示后强制附带免误读声明：`文风示例仅供表达方式参考，不为正文具体内容提供情节借鉴。`
- **偏好短提示**：Gemini 对更短更直接的提示响应更好。
- **强制 few-shot 示例**：总是包含 `>` 示例——Gemini 标签理解差，但示例理解强，示例能压制其自由发挥空间。

### 3.3 标签连锁机制

Gemini 对标签的拆解存在严重的极端化问题——它不是不理解标签，而是把一个多样的、因人而异的标签拆成一串单一、高频、八股的子行为，然后把这些东西塞回正文。

- **标签拆解链条**：你输入一个提示词 → Gemini 自动连锁与之相关的高频区 → 高频区内容互相触发 → 正文八股化。
- **"羞耻"案例**：Claude/GPT 知道不同性格的人对羞耻反应完全不同；Gemini 只给两种：脸红和崩溃。"害羞""娇羞"同样存在极端化倾向。
- **行为序列翻译法**：将性格标签翻译为具体的行为序列代替。如"羞耻"→ "延迟回复、话题转移、截断语法、自我保护姿态"。参照 `label-risk-table.md`。

### 3.4 高频区规避

每个 AI 模型都有自己的高频区——高频区的内容特别容易被连锁触发。

- **高频区定义**：模型训练数据中出现频率极高的表达模式，一旦被提示词触发就自动播放。
- **Gemini 高频区识别**：当正文中出现大量你在提示词中没有要求的表达时，说明触发了高频区。
- **精简是最好的规避**：提示词越长，踩中高频区的概率越高。
- **低频同义替换**：用不那么常见的同义表达替换高频词汇（如：不用"温柔"而用具体关怀动作描述）。

## 4. 跨模型通用原则

### 4.1 U 形注意力分配

LLM 对上下文的注意力呈 U 形曲线（Liu et al. 2024）：开头和结尾的信息准确率最高，中间段准确率下降超过 30%。

- 将核心铁律和控制边界放在 `<control_scope>`（开头）和 `<prompt_discipline>`（结尾）。
- 避免将最重要的规则埋在 `<expression>` 或 `<aesthetics>` 的中部。
- 重置规则放在 `<prompt_discipline>` 末尾，利用尾部注意力优势。

### 4.2 正面表述原则

"只使用真实数据"优于"不要使用虚假数据"（粉象效应）。

- 积极提示词 + 消极提示词结合效果最好。
- 写法优先级：`你要这么写且你不能那样写` > `你要这么写` > `你不能那样写`。
- 每条负向约束后以 `→` 搭配正向替代动作。

### 4.3 高频区通用防御

所有模型都有高频区，但严重程度不同（Gemini >> Claude > GPT）。

- 提示词越精简，踩中高频区的概率越低。
- 用一个精选示例替代三句模糊描述——示例占用 token 但不触发高频区，描述性规则容易触发。
- 检查提示词中的词汇是否在目标模型的生成文本中异常频繁出现——若是，替换为低频同义词。
