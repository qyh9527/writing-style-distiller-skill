## 1. 总体结构

最终输出必须由 `<writing_style>` 标签包裹。标签属性包括：
- `author`：作者名或风格名
- `source`：`author` / `text` / `samples` / `synthetic`
- `mode`：`style-only` / `style-with-affective-baseline`
- `family`：`情感表达` / `语言特征` / `叙事结构` / `示例驱动` / `场景响应` / `对话驱动`

### 1.1 分区位置优化原则

基于 U 形注意力曲线（Liu et al. 2024）：LLM 对上下文窗口的**开头和结尾**注意力最强，中段准确率下降 30%+。分区排列应利用这一特性：

- **头部高注意力区**：`<control_scope>` 必须占据首位，建立控制边界，防止文风示例被误读为剧情指令。
- **中段承载区**：`<narrative>` → `<expression>` → `<aesthetics>` → `<dynamics>`（可选）依次排列，承载风格主体内容。
- **尾部高注意力区**：`<prompt_discipline>` 必须放在末位，其中的铁律与负向约束利用尾部注意力优势获得最强执行力。

这一排列顺序在所有文风配置中固定不变。

## 2. 一级分区标签

标签内只允许以下 5 个一级 XML 分区标签，分区内部必须为纯文本：

1. `<control_scope>` — 控制边界声明：文风与基调/人设的界限。标准内容必须包含"示例仅供参考"声明，模板如下：
   ```
   本配置控制措辞、句法、叙事、段落、节奏、对话处理与感官选择
   不控制剧情走向、人物最终结局与角色具体决策
   示例仅引导语言表达，不作为剧情情节的借用指令
   文风示例仅供思路参考，不为任何具体剧情提供写作借鉴
   ```
2. `<narrative>` — 叙事系统：人称聚焦、标记词、时序、节奏、段落控制。
3. `<expression>` — 表达系统：描写风格、对白模式、人物塑造、感官层级。
4. `<aesthetics>` — 美学系统：核心概念、语言特征、美学目标。
5. `<prompt_discipline>` — 规训系统：正向指令、负向约束、标签风险、模型适配、重置规则。
6. `<dynamics>`（可选） — 动态系统：状态分级、条件触发规则、温度层/称呼升级引擎、机制间交互。

### `<dynamics>` 使用指南

`<dynamics>` 是**可选的第六区标签**，仅在以下情况下启用：

- **场景响应类文风**：需要定义状态分级表、触发条件映射、机制间交互规则
- **对话驱动类文风**：需要定义称呼升级引擎、温度层切换系统、声口变化规则
- **需要在 narrative/expression 中放不下的动态切换逻辑**

不启用 `<dynamics>` 的情况：
- 情感表达类、叙事结构类、示例驱动类文风通常不需要
- 语言特征类文风的语域规则已由 `<aesthetics>` 承载
- 规则可以自然归入现有五区的，不应为了使用新标签而拆分

### 格式扩展说明

- **场景节奏循环模板**：在 `<dynamics>` 或 `<narrative>` 中，可使用场景节奏循环格式定义叙事推进节拍，如「身临其境→言笑晏晏→切肤之亲→循环往复」。每个节拍用一句话说明其叙事功能。
- **"底色"声明**：当文风包含强烈的语言风格（如骚话、毒舌）时，可在 `<control_scope>` 中增加底色声明，明确该风格不改变角色人格，仅影响措辞表层。
- **变量封装**（SillyTavern 生态可选）：复杂机制系统（如 NSFW 现实化指导、状态分级表）可使用 `{{setvar::}}` 变量封装，将大段规则存入变量以减少 prompt 主体长度。此为平台特定的可选优化，非必须格式。

## 3. 内容格式规范

- **规则条目**：使用「条目名：规则内容」格式书写。
- **并列属性**：使用 `属性=值 | 属性=值` 格式紧凑排列。
- **原文示例**：用 `>` 开头标记原文，紧跟所属规则下方。长度 30-80 字符。保持原语种。
- **模型特定规则**：用 `[模型名]` 方括号标记，如 `模型适配[Claude]：...`
- **禁止事项**：不使用任何 XML 子标签嵌套。一级分区标签内部全部是纯文本。

## 4. 可选独立 `<tone>` 块

仅在用户明确请求剧情基调时，在 `<writing_style>` 之后输出。基调应当**尽量精简**，不用很多字。

从详细到极简的梯度示例：

**治愈基调（标准格式）：**
```
<tone name="治愈">
基线：温暖安定，允许微小波折但情绪底色始终柔软
事件处理：负面事件以消化后的释然收束，正面事件用日常细节放大幸福感
情感逻辑：向暖收敛
边界声明：此块控制剧情基调，非措辞
</tone>
```

**伤感基调（精简格式）：**
```
<tone name="伤感">
基线：低沉绵长，美好事物带着即将失去的预感
事件处理：正面事件被回忆滤镜染上惆怅，负面事件沉入长久沉默
情感逻辑：向沉收敛
</tone>
```

**消极基调（极简格式）：**
```
<tone name="消极">
基线：倦怠灰暗，一切热情被钝化
情感逻辑：向冷收敛，正面事件引发怀疑而非欣喜
</tone>
```

## 5. 完整示例：汪曾祺风格

```xml
<writing_style author="汪曾祺" source="author" mode="style-only" family="语言特征">

<control_scope>
本配置控制措辞、句法、叙事、段落、节奏、对话处理与感官选择
不控制剧情走向、人物最终结局与角色具体决策
示例仅引导语言表达，不作为剧情情节的借用指令
</control_scope>

<narrative>
人称与聚焦：第三人称，限制性外聚焦为主，叙述距离适远，保持旁观叙史感
标记词：且说、话说、便、因、原来、倒
时序：绝对主干顺叙，偶尔使用插叙交代背景家世
节奏：松弛舒缓，主干情节推进快速，在市井生活、器物、服饰处长停顿
  > 过了一个湖。好大一个湖！穿过一个县城。
段落控制：
  单位=场景拍 | 长度=三五句构成的小自然段，疏朗开阔
  对话分离=对白一律独立成段，不带任何对话神态标签
  转换=情感沉静或介绍地方特产时段落拉长，轻快拌嘴或动作骤变时段落极细碎
  > 明子听见有人跟他说话，是那个女孩子。
  > "是你要到荸荠庵当和尚吗？"
  > 明子点点头。
</narrative>

<expression>
描写：白描为主，不用比喻，遣词追求准确、畅通、平易、雅致，大巧不工
  > 县城真热闹：官盐店，税务局，肉铺里挂着成爿的猪肉
对白：半明清近古白话，一人一句错落排开，依靠对答内容直接区分角色
  > "当和尚要烧戒疤！你不怕？"
  > 明子不知道怎么回答，就含含糊糊地摇了摇头。
人物塑造：通过白描日常习惯与代表动作体现内心性格，不罗列心理细节，拒绝揣测非聚焦角色心思
感官层级：视觉 > 嗅觉 > 听觉
</expression>

<aesthetics>
核心：淡雅 / 市井烟火气 / 散文化 / 随遇而安 / 俗中见雅
语言：
  句法=老派白话文，流水句推进，多短散句，偶有四六对仗句点睛
  词汇=平实口语化动词，适量带有元明清地方俚俗，拒绝高冷成语与纯文言生僻词
  修辞=白描为主，禁用虚浮不着调的比喻
美学目标：写尽中国平凡生活的温热，不将平淡生活宏大化，以文学口语带给读者干净、治愈却坚韧的美学体验
</aesthetics>

<prompt_discipline>
正向指令：使用简短明快的流水短句。叙事中穿插景物服饰，作为渲染情绪的无声留白。
负向约束：禁用对话标签（如"他低声说道"）→ 改用纯动作转场
标签风险：温柔→具体关怀动作（如剥莲蓬、改衣服）代口头抒情
模型适配[Claude]：抑制过于精致的长叹与宿命感抒情，保持市井白话的平实感
重置规则：走出 NSFW 或大情绪场景后，用清爽市井日常描写和独立短句重置本配置基线
</prompt_discipline>

</writing_style>
```
